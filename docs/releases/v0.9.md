# v0.9 Streamlit App Testing and UX Reliability

v0.9 improves app-level reliability for GitHub demos and resume presentation. It does not add new causal estimators or replace the deterministic causal pipeline.

## Highlights

- Added Streamlit app smoke tests using Streamlit AppTest when available.
- Added demo-mode regression checks for public demo safety messaging.
- Added UI reliability checks for key result sections such as Data Quality, Causal Trust Summary, Robustness / Sensitivity Notes, Heterogeneity Explanation, LangGraph trace/copy, and report downloads.
- Preserved deterministic causal pipeline and optional dependency fallback.
- No real data, API keys, database, login, or production deployment added.
- Updated README, CHANGELOG, AGENTS, demo script, and interview notes.
- Full test result: `82 passed, 1 skipped` with `python -m pytest -q`.

## Testing Scope

The new smoke tests use `streamlit.testing.v1.AppTest` when available and skip gracefully if Streamlit does not provide the testing module. The tests avoid brittle screenshots, real browser automation, network access, API keys, file uploads, and private local files.

Covered paths:

- Initial Streamlit page render without uncaught exceptions.
- Public demo mode safety copy.
- Optional dependency status panel rendering when optional dependencies are unavailable.
- Default sample-data analysis path through the result display.
- Key result sections for Data Quality, Causal Trust, Sensitivity, Heterogeneity, LangGraph copy, and Markdown / HTML report downloads.
