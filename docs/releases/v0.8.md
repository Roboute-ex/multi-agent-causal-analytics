# v0.8 Causal Robustness and Interpretability

v0.8 focuses on making causal analysis results easier to trust, explain, and maintain. It does not add another orchestration framework and does not replace the deterministic pipeline.

## What Changed

- Added Causal Trust Summary to translate ATE, refutation status, reviewer warnings, data quality warnings, sensitivity notes, and heterogeneity signals into a conservative trust summary.
- Added Robustness / Sensitivity Notes to summarize whether existing refutations keep a stable direction.
- Added Heterogeneity Explanation to make optional CATE output more useful for business segmentation discussions.
- Added the three summaries to Streamlit after a run completes.
- Added the three summaries to Markdown and HTML report export.
- Added a low-cost optional PDF summary when ReportLab is available.
- Added tests for the new services and report export behavior.

## Design Boundaries

- Deterministic orchestration remains the default stable path.
- LangGraph remains optional and experimental.
- DoWhy, EconML, DeepSeek, SHAP, LangGraph, and ReportLab are not required for the new v0.8 tests.
- No OpenAI API is used.
- No database, login system, deployment configuration, or uploaded-data persistence is added.
- No real business data should be committed.

## Core Files Intentionally Not Changed

- `app/core/orchestrator.py`
- `app/core/schemas.py`
- `app/services/causal_dowhy.py`
- `app/services/cate_econml.py`
- `app/services/variable_recommender.py`
- `app/graph/langgraph_runner.py`

## Interpretation Guidance

The new summaries are intentionally conservative. They should be presented as robustness and interpretability aids, not as proof of causal identification. Observational data can still suffer from unmeasured confounding, sample quality issues, measurement error, and invalid variable choices.

## Suggested Demo Talking Point

In v0.8, the project shifts from adding more visible framework features to improving causal result trustworthiness. The app now turns ATE, refutation checks, data quality diagnostics, and optional CATE output into plain-language warnings and recommendations.
